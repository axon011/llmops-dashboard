# .agent.md - llmops-dashboard

## Project Info

AI Coding Usage Analytics Dashboard - Track and visualize OpenCode usage with token metrics, model breakdown, and daily usage trends.

## Key Technologies
- React 18 + TypeScript (frontend)
- FastAPI (backend)
- SQLite (reads from OpenCode database)
- PostgreSQL (session storage)
- TailwindCSS, Recharts (visualization)

## Directory Structure

```
llmops-dashboard/
├── app/
│   ├── main.py              # FastAPI entrypoint
│   └── routes/
│       ├── chat.py          # /chat endpoint
│       ├── metrics.py       # /metrics endpoints (mock Langfuse data)
│       └── opencode.py      # /opencode endpoints (real OpenCode data)
├── frontend/
│   └── src/
│       └── App.tsx         # React dashboard with tabs
├── tests/
│   └── test_routes.py
├── docker-compose.yml
├── Dockerfile
└── requirements.txt
```

## Key Features
- **OpenCode Usage Tab** - Real token usage from your OpenCode SQLite database
- **Langfuse Metrics Tab** - Demo/mock metrics for observability visualization
- Provider breakdown: opencode, github-copilot, zai-coding-plan
- Token analysis: input, output, reasoning, cache

## Key Endpoints
- `GET /opencode/metrics` - Real OpenCode usage metrics (reads from SQLite)
- `GET /metrics/sessions` - Mock Langfuse session metrics
- `POST /chat` - Chat with LLM (requires API keys)
- `GET /health` - Health check

## Running the Project
```bash
# Build and run
docker build -t llmops-dashboard-app .
docker run -d --name llmops-dashboard-app -p 8000:8000 --env-file .env -v "C:/Users/Aravind/.local/share/opencode:/opencode_data:ro" llmops-dashboard-app

# Or with docker-compose
docker-compose up -d --build
```

## How to Identify This Project
- **Keywords**: OpenCode, usage tracking, token metrics, dashboard
- **Unique feature**: Reads real data from OpenCode SQLite database
- **NOT**: This is NOT multi-agent-pipeline (that's LangGraph/CrewAI) or rag-eval-system (that's RAG/Qdrant)
